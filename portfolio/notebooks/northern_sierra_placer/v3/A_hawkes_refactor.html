v3 Phase A: Hawkes cell-mask refactor

Verifying the per-fold-Hawkes optimization that unlocks v3’s iteration speed

Author

Sky King

Published

June 2, 2026

What this notebook is

v3 Phase A.1 added a cell_mask parameter to hawkes_dual_decay_catchment so the per-fold recompute in train_predict_250m.py skips the ~45% of grid cells outside the AOI. The v2 hot loop iterated over the full bounding-rectangle grid (~1.4 M cells) then discarded everything outside the AOI; v3 only visits the AOI-clipped cells (~800 k).

This notebook verifies the refactor’s correctness on a tiny synthetic problem and on the actual placer grid, and measures the wall-clock speedup so we know whether to keep going with v3.

1. Equivalence on the synthetic 10×10 problem

The unit-test fixture (tests/test_hawkes_cell_mask.py): a 10×10 grid, 5 samples along a single NHD reach, one element column. Run the function with no mask (compute every cell), then with a mask that includes only the first 30 cells. Verify the masked-true cells are numerically identical to the unmasked call, and the masked-false cells are NaN.

Code 
xs = np.arange(-50.0, 950.0, 100.0)
ys = np.arange(-50.0, 950.0, 100.0)
grid = Grid(xs=xs, ys=ys, resolution_m=100, crs="EPSG:3310")

nhd = gpd.GeoDataFrame(
    {"comid": [1001], "arbolate_sum": [10.0], "hydroseq": [50.0],
     "geometry": [LineString([(50, 50), (950, 950)])]},
    crs="EPSG:3310",
)
samples = gpd.GeoDataFrame(
    {"Au_ppm": [0.5, 1.0, 2.0, 3.0, 5.0],
     "geometry": [Point(100 + 150 * i, 100 + 150 * i) for i in range(5)]},
    crs="EPSG:3310",
)

unmasked = hawkes_dual_decay_catchment(samples, nhd, grid, element="Au_ppm").to_numpy()

cell_mask = np.zeros(grid.n_cells, dtype=bool)

masked = hawkes_dual_decay_catchment(
    samples, nhd, grid, element="Au_ppm", cell_mask=cell_mask,

print(f"mask-false cells that are NaN:      "
      f"{int(np.isnan(masked[~cell_mask]).all())} (all of {int((~cell_mask).sum())})")

3. Timing benchmark (synthetic grid, repeated runs)

The synthetic 10×10 problem is too small to show the speedup at scale (cell-loop overhead is dominated by setup costs at that size). For a more representative benchmark, run the unmasked vs masked version 50 times each on the synthetic problem and report the ratio.

Code 
def time_runs(fn, n=50):
    fn()  # warmup
    t0 = time.perf_counter()
    for _ in range(n):
        fn()
    return (time.perf_counter() - t0) / n

t_unmasked = time_runs(
    lambda: hawkes_dual_decay_catchment(samples, nhd, grid, element="Au_ppm")
)

cell_mask_10pct = np.zeros(grid.n_cells, dtype=bool)
cell_mask_10pct[::10] = True
t_masked = time_runs(
    lambda: hawkes_dual_decay_catchment(
        samples, nhd, grid, element="Au_ppm", cell_mask=cell_mask_10pct,
    )
)

print(f"unmasked (all 100 cells):  {t_unmasked*1000:.3f} ms/call")
print(f"masked (10 cells):         {t_masked*1000:.3f} ms/call")
print(f"speedup ratio:             {t_unmasked/t_masked:.2f}x")
print()
print("Note: synthetic grid is 100 cells; the AOI placer grid is ~1.4M cells,")
print("so the per-fold speedup on the real workload should be much larger")
print("(closer to 1.8x given 45% of the grid is outside the AOI mask).")

4. Expected wall-clock impact on v3 training

The v2 per-fold-Hawkes runs cost roughly:

  • LightGBM CV: 156 min for 53 folds × 3 elements = 159 hawkes calls
  • Stacking OOF (Tertiary): 440 min for 150 fold × 3 elements = 450 calls

Most of that time is the hawkes hot loop. With 45% of cells skipped per call, the expected wall-clock drops to ~85 min (LGBM CV) and ~240 min (Stacking OOF) in v3 — a saving of ~50% of the per-fold hawkes wall clock.

Total v3 training expectation (per the v3 plan, both populations): ~5-6 hours instead of v2’s ~16+ hours. Other v3 changes (XGBoost addition, per-population feature splits) add work, but Hawkes was the dominant cost.
