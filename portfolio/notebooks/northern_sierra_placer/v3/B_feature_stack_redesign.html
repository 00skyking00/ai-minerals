What this notebook is

v3 Phase B redesigned the feature engineering in five sub-phases (B.0, B.0a, B.1+B.4, B.2+B.5, B.6). This notebook shows:

  • The per-population feature lists and weight dicts as they ship in v3
  • Maps of the three new B.6 features (plan/profile curvature, lithological contact distance)
  • Map of the B.2 buffered hydraulic-pit-proximity feature against the unbuffered v2 version, showing the NaN halo around each pit

No model retraining happens here. The v3 retrain runs at end of Phase F.

1. Per-population feature stacks (B.1 + B.4)

The Quaternary-only features are the modern-channel signal: spi_band (Roy 2018 detachment threshold), ksn (knickpoint steepness), flow_acc cluster, and the Hawkes catchment-As + catchment-Sb (pathfinders that don’t help with Tertiary buried channels).

2. Per-population scorer weights (B.1 + B.4)

The Phase 1 scorer also splits. The v2 single weight set is preserved as DEFAULT_WEIGHTS for backward compatibility; v3 ships DEFAULT_WEIGHTS_TERTIARY and DEFAULT_WEIGHTS_QUATERNARY for the gated per-population score.

    rows = []

3. Soft-falloff vs hard-cap (B.5)

The v2 distance features used direction="inverse" with a hard cap (cap=15_000 for lode distance, cap=25_000 for downstream). Cells beyond the cap got score 0; cells at the boundary got an arbitrary near-zero score. The transition was a visible discontinuity in the score maps.

v3 uses direction="gaussian" with a sigma_m parameter. The score decays smoothly from 1 (at distance 0) to ~5% at distance 3σ. Forest Hill is ~12 km from the nearest lode-Au point; v2’s 15km cap put it at score ~0.2; v3’s gaussian with σ=12km puts it at score ~0.61, which is much more in line with the geological reality (Forest Hill is a real placer district).

Code 
def hard_cap_inverse(d, cap):
    out = np.where(d < cap, 1.0 - d / cap, 0.0)
    return out.astype(np.float32)

d_km = np.linspace(0, 30, 200)
d_m = d_km * 1000

fig, ax = plt.subplots(figsize=(9, 4.5))
ax.plot(d_km, hard_cap_inverse(d_m, 15_000), color="crimson",
        linewidth=2, label="v2: inverse, cap=15km")
ax.plot(d_km, gaussian_falloff(d_m, sigma_m=12_000),
        color="steelblue", linewidth=2, label="v3: gaussian, σ=12km")
ax.axvline(12, color="gray", linestyle=":", alpha=0.6, label="Forest Hill: ~12km from nearest lode-Au")
ax.set_xlabel("distance to lode-Au point (km)")
ax.set_ylabel("feature contribution (0 = no signal, 1 = max)")
ax.set_title("distance_to_lode_m: v2 hard cap vs v3 gaussian falloff")
ax.legend()
ax.grid(True, alpha=0.25)
plt.tight_layout()
plt.show()

4. Buffered hydraulic-pit-proximity (B.2)

The v2 hydraulic_pit_proximity_m feature trivially encoded the Tertiary label: positive cells are pit polygon centroids, so distance == 0 perfectly identifies them. v3 trains on hydraulic_pit_proximity_m_buffered, which is NaN within 1km of any pit polygon. The model cannot learn the shortcut because the positive cells themselves are unobservable in this feature.

Compute both maps on a small AOI patch around the anchor cluster + plot the buffer halo.

Code 
from ai_minerals.grid import build_grid
from ai_minerals.data.adapters import get_adapter

    REGION.raw_paths["hydraulic_pits"], REGION.aoi

prox_v3 = hydraulic_pit_proximity_m_buffered(pit_polys, grid, buffer_m=1000.0)

The buffered version voids the ~10-20k cells closest to pit polygons. The model’s view of “near a pit” is replaced by “I have no signal here” — which forces the model to use the other features (geomorphology, geology, geochem) to predict positives. That’s exactly what v3 needs for generalization.

5. Three new B.6 features

Plan and profile curvature

The DEM second-derivative curvatures, Gaussian-smoothed.

  • Plan curvature: positive on ridges (interstream divides where Tertiary gravels rest), negative in valleys. Strong signal for Tertiary terrace identification.
  • Profile curvature: positive at convex slope-breaks (knickpoints — gold traps for Quaternary placers), negative at concave breaks (bench toes — also placer traps).

Both functions take a DEM array; the assemble script wires them in at the grid level alongside slope and TPI.

Code 
# Compute on a synthetic DEM patch to show the maps are reasonable.
nx, ny = 200, 200
xs = np.linspace(-1, 1, nx)
ys = np.linspace(-1, 1, ny)
X, Y = np.meshgrid(xs, ys)
dem = 1000 + 200 * np.sin(3 * X) * np.cos(2 * Y) + 50 * (X**2 + Y**2)

fig, axes = plt.subplots(1, 3, figsize=(15, 5))
im0 = axes[0].imshow(dem, cmap="terrain")
axes[0].set_title("synthetic DEM\n(elevation, m)")

Distance to lithological contact

Per-cell distance to the nearest CGS 2010 polygon boundary. Contact zones favor lode → placer transport (fluids exit the source rock at the contact, metals concentrate there).

Code

6. What’s deferred to v4

  • HUC12 categorical feature (planned in B.6): the NHDPlus HR GeoPackage ships flowlines but not WBD HUC12 polygons. v4 should add a separate WBD download adapter.
  • Per-population paleochannel composite (planned in B.3): v3 keeps the single composite. Per-population recompute would require an additional precompute pass and is deferred.
  • Paleochannel orientation feature: ~3 hours to build but uncertain marginal signal. v4 candidate.

What’s next (Phase C)

Three model upgrades:

  1. XGBoost added to the stacking pool (RF + LightGBM + XGBoost).
  2. nnPU for the Quaternary branch (modern positive-unlabeled method replacing the 2014 Mordelet-Vert PU bagging).
  3. Calibrated logistic stacking for the two-population fusion, replacing the per-cell np.maximum operator.

Then Phase D (data additions), Phase E (validation upgrades), Phase F (full retrain + audit + ship).
