What this notebook is

Phase C added three model-side upgrades that ship in v3 regardless of whether Phase F.1 retrain has run yet:

  1. C.1 XGBoost as a third base learner alongside Random Forest and LightGBM. Stacking pool widens from 2 → 3 base models.
  2. C.2 nnPU for the Quaternary branch only. Replaces the 2014 Mordelet-Vert PU bagging with the Kiryo 2017 non-negative risk estimator.
  3. C.3 Calibrated logistic stacking as an alternative to the v2 per-cell np.maximum fusion. Both modes ship in v3; logistic is the new default.

This notebook verifies the implementation (smoke imports, factory defaults, loss curve on synthetic data, fusion comparison) but does NOT retrain v3. The full v3 retrain happens at Phase F.1.

C.1 XGBoost defaults

XGBoost is the most-cited tree learner in 2024-2025 MPM comparison papers and the v3 stacking pool’s third base model. The defaults match RF + LightGBM in spirit: medium-depth trees, learning-rate-style boosting, native NaN routing.

The hist tree method is the modern default (histogram-based split finding, 5-10x faster than the legacy exact method on tabular data). Native NaN routing means NaN values move to whichever split-direction produces lower training loss; we keep the -9999.0 sentinel fill in train_predict_250m.py because most upstream features are already NaN-cleaned by the assemble step.

C.2 nnPU loss on a synthetic placer-shaped problem

The Mordelet-Vert PU bagging in model_pu.py draws N “negative” subsamples from the unlabeled set, trains a classifier per subsample treating those as negatives, and averages. With 573 Quaternary positives in 800k cells (0.07% positive rate), the unbiased PU loss can go negative when the negative-class loss estimate exceeds the positive-class loss. Kiryo 2017’s nnPU clamps that negative-risk term at zero, which prevents the over-fitting failure mode at low positive rates.

Demo: synthesize a small placer-shaped problem (1000 cells, 10 features, 20 positives, signal feature linearly separates), train nnPU for a few epochs, plot the predicted-probability distribution split by true label.

Code 
np.random.seed(42)
n_cells = 1000
n_feat = 10
n_pos = 20
X = np.random.randn(n_cells, n_feat).astype(np.float32)
# Signal feature: positive cells have larger value in column 0
y = np.zeros(n_cells, dtype=np.float32)
pos_idx = np.random.choice(n_cells, n_pos, replace=False)
y[pos_idx] = 1.0
X[pos_idx, 0] += 2.5  # signal in feature 0

df = pd.DataFrame(X, columns=[f"f{i}" for i in range(n_feat)])
df["is_pos"] = y
feature_cols = [f"f{i}" for i in range(n_feat)]

proba, _ = fit_nnpu_quaternary(
    df, label_col="is_pos", feature_cols=feature_cols,
    prior=0.02, n_epochs=30, batch_size=64, random_state=42,
)

print(f"nnPU output shape: {proba.shape}")
print(f"  proba range: [{proba.min():.4f}, {proba.max():.4f}]")
print(f"  median proba at positives:   {np.median(proba[y == 1]):.4f}")
print(f"  median proba at unlabeled:   {np.median(proba[y == 0]):.4f}")

fig, ax = plt.subplots(figsize=(9, 4.5))
ax.hist(proba[y == 1], bins=30, alpha=0.6, color="crimson", label=f"positives (n={int(y.sum())})")
ax.hist(proba[y == 0], bins=30, alpha=0.6, color="steelblue",
        label=f"unlabeled (n={int((y == 0).sum())})")
ax.set_xlabel("nnPU predicted probability")
ax.set_ylabel("count")
ax.set_title("nnPU on synthetic placer-shaped problem (1000 cells, 20 positives, prior=2%)")
ax.legend()
ax.grid(True, alpha=0.25)
plt.tight_layout()
plt.show()

Even on 30 epochs and a noisy 10-feature problem, nnPU separates the positive cells from the unlabeled bulk. On the real placer problem (800k cells, 573 positives, 18 features, prior~0.07%) the separation is what v3 Phase F.1 will measure.

C.3 Calibrated logistic fusion vs np.maximum

v2 fused the two per-population calibrated rasters with np.maximum. For each cell, the higher of the two probabilities wins. Simple, but it can over-state prospectivity when both populations are weakly-positive at a cell (e.g., both emit 0.0004 and the max takes that as the final answer).

v3 ships a second fusion mode: a sklearn LogisticRegression trained on (p_cal_t, p_cal_q) plus context features (distance_downstream_from_lode_m, is_quaternary_alluvium) against the union of positive labels. The meta learns how to weight the two populations based on geomorphic context: in Quaternary alluvium the meta should up-weight p_cal_q; outside Qa it should up-weight p_cal_t.

Since v3 Quaternary hasn’t been trained yet, we can’t run the comparison against real per-population calibrated rasters. Demonstration here uses the v2 Tertiary calibrated parquet only; the np.max degrades to “use Tertiary when Quaternary is missing” and the logistic mode warns and skips. The full A/B happens in Phase F.
